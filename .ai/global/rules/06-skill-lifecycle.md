# Skill Lifecycle Policy

- Create a new skill only when the workflow is repeated at least 3 times or is high-risk and needs deterministic steps.
- Do not create a skill for one-off tasks; use regular task instructions instead.
- Keep each skill focused on one repeatable workflow and avoid mixing unrelated domains.
- Do not duplicate rule content inside skills; skills should reference canonical rules where possible.
- Review skills periodically and remove stale skills that are no longer used.
